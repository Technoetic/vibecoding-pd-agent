# 코딩 한 줄 몰라도 됩니다. 단, AI에게 '이것' 안 시키면 무조건 실패합니다.  `[승인대기]`

> task_id: CONTENT_2026_006 · 주제: AI한테 코딩 시킬 때 초보가 가장 많이 하는 실수 3가지

## 🎬 스크립트
AI한테 코딩 시키는데 결과가 엉망인가요? 대부분의 비개발자가 범하는 치명적인 실수 3가지를 짚어드립니다. 첫째, '알아서 다 해줘'식의 뭉뚱그린 프롬프트입니다. AI는 훌륭한 비서지만, 구체적인 설계도 없이는 길을 잃습니다. 원하는 기능과 데이터 흐름을 명확히 정의하세요. 둘째, 단계별 검증을 건너뛰는 것입니다. 한 번에 모든 코드를 짜달라고 하지 마세요. 작은 단위로 나눠서 에러를 잡아야 합니다. 셋째, AI가 짠 코드를 맹신하는 것입니다. 로직이 맞는지 반드시 직접 흐름을 확인해야 사고를 방지합니다. 바이브코딩은 코딩을 배우는 게 아니라, AI라는 도구를 지휘하는 능력입니다. 이 3가지만 기억해도 작업 효율이 완전히 달라질 겁니다. 지금 바로 이 원칙을 적용해 웹서비스 런칭을 시도해볼 수 있습니다. 더 자세한 방법이 궁금하다면 채널의 고정 댓글을 확인해보세요.

## 🖼 스토리보드
1. 0-3초: 훅 구간. 당황하는 사람의 얼굴과 '코딩 엉망?' 자막 오버레이
2. 3-15초: 1번 실수 설명. 추상적인 프롬프트 vs 구체적인 설계도 비교 화면
3. 15-30초: 2번 실수 설명. 쪼개서 질문하는 모습과 에러 해결 과정 시각화
4. 30-45초: 3번 실수 설명. 코드 검토하는 모습. '맹신 금지' 경고 아이콘
5. 45-60초: 마무리. 바이브코딩 철학 강조 및 고정 댓글 CTA
