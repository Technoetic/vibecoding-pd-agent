# 포트폴리오 — AI 콘텐츠 PD 에이전트 산출물 큐레이션

> 채용 과제 제출용(합격 가이드 §5.1). 4-에이전트 오케스트레이터가 **실제로 생산·검수·승인**한 쇼츠 기획안 중 대표 5건. 각 기획안이 **양실장 바이브코딩대학 채널의 성격**(그로스해커·수익화·비개발자 타깃·SME 블루오션)을 어떻게 반영했는지 주석을 달았다. 전부 Eval 검수(채널중복·독창성·한국 수익규제·완주율 등) 통과 + `[승인대기]`(인간 승인 전).

**선별 기준:** 채널 중복(자카드 ≥0.6) 산출물은 제외(001·007 — 채널에 이미 있는 "개발 없이 5일" 주제). 도메인 다양성(앱·SME·교육·로컬·리뷰) + 검수 품질로 선별.

## 1. CONTENT_2026_009 — "비개발자가 AI로 사장님 가게 리뷰 자동 응대 만들기" ⭐ 대표작

- **eval 12종 전부 통과** (가장 최신·완전한 검수). originality 0.2026·channel-dup 0.25·실시간 트렌드 4건 반영.
- **채널 반영:** 양실장의 **SME(중소상공인) 마이크로 디지털 전환(MDX)** 블루오션 직격 — "혁신 앱"이 아니라 "리뷰 자동 응대"라는 구체적 SME 페인포인트. 그로스해커형("월 100만원 손해" = 손실 회피 심리)이되 수익규제 통과(가능성 프레이밍).
- **차별점:** 채널 중복 검사가 양실장 실제 영상("PD 지원자 과제")과 대조해 distinct 확인 → "채널이 안 다룬 새 주제"임이 결정론으로 입증. → [[SESSION_LOG]]

## 2. CONTENT_2026_008 — "AI로 동네 카페 단골 만드는 인스타 자동화 비법"

- **originality 0.0937**(7건 중 최저 = 가장 독창적). 실시간 트렌드 5건+출처 8개 반영.
- **채널 반영:** 로컬 비즈니스(동네 카페) + 자동화 = SME 타깃. "DM 답장 3초 컷, 예약 폭주"는 그로스해커 지표(전환) 지향. Trend Analyst가 Sonar로 "에이전틱 AI·n8n/Make 자동화" 트렌드를 흡수해 카페 맥락에 연결.
- **차별점:** 데모 종단 검증의 산출물 — 트렌드→키워드→스크립트 파이프라인이 라이브로 작동한 증거.

## 3. CONTENT_2026_006 — "코딩 한 줄 몰라도 됩니다. 단, AI에게 '이것' 안 시키면 무조건 실패합니다."

- 시간분절 스토리보드(0-3초/3-15초…)·실수 3가지 구조. **정체성형 훅**(가장 안전·강력).
- **채널 반영:** 양실장 **"바이브코더 정체성"**·"코딩보다 AI 지휘 능력"과 정합. "AI라는 도구를 지휘하는 능력"은 채널 교육철학('뿌리지식') 직결.
- **차별점:** Reviewer가 1차 반려("오늘 바로 런칭 가능"=수익 단정)→재작성("런칭을 시도해볼 수 있습니다"=가능성)으로 **한국 표시광고법 컴플라이언스가 작동**한 사례.

## 4. CONTENT_2026_005 — "코딩 없이 3분 만에! 사장님 가게 예약 시스템 만들기"

- **채널 반영:** SME 페인포인트 직격("전화 응대·예약 누락")·ROI 영업 앵글. 양실장 채널의 "혁신 앱 아닌 구체적 고통점 해소"(버티컬 AI) 정합.
- **차별점:** G1(originality 코사인 유사도 실구현) 종단 검증 산출물 — 실측 0.1297로 독창성 통과.

## 5. CONTENT_2026_003 — "AI로 비개발자가 만든 첫 앱, 무엇이 달랐나?"

- **질문형 훅**(정보 공백 이론 — 결론 숨김). "3주 만에" 초단기 시간 강조형 + "비개발자" 정체성형 결합.
- **채널 반영:** 비개발자 성공 서사 = 채널 핵심 내러티브("바이브 프리랜서 두 달 3,500만 원" 류). 단 수익 수치는 역U자 법칙 따라 절제.
