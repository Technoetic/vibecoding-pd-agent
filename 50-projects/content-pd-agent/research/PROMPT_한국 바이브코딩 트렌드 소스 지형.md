# 딥리서치 프롬프트 — 한국 바이브코딩 트렌드 소스 지형

## 사용법
이 노트는 메타 정보다. **외부 딥리서치 도구(Perplexity/Gemini/ChatGPT/Claude.ai Deep Research)에 넣을 입력본은 페어 파일** `PROMPT_한국 바이브코딩 트렌드 소스 지형.deepresearch.txt`를 그대로 복사해 쓴다.

## 결손 영역 (현 상태 vs 결손)

| 항목 | 현 상태 | 결손 |
| :-- | :-- | :-- |
| 경쟁 채널 벤치마크 | [[한국 AI 교육 유튜브 시장 구조]]에 4채널(조코딩·장피엠·양실장·에어빌드) 포지션·포맷·훅 | 트렌드 정보 **소스 계층**(1차 담론 포럼 → 번역 미디어 → 튜토리얼 채널)의 생태계 지형 부재 |
| Trend Analyst 입력 | Sonar 실시간 트렌드 다단계 조사(ADR-021) | "어떤 소스가 한국 비개발자 향 주제를 가장 빨리 보여주는가"의 **소스 신뢰도·신속도 맵** 부재 |
| 채널 차별화 근거 | 양실장 "후발 진정성·프리랜서 현실" 포지션 | 같은 니치(비개발자·노코드·자동화)에서 **양실장과 직접 경쟁/인접한 한국 채널** 추가 발굴 부재 |

## 왜 이게 콘텐츠 기획에 유용한가
- Trend Analyst가 "지금 뜨는 주제"를 더 정확히 길어올리려면, 단발 검색을 넘어 **어떤 소스가 한국 비개발자 향 트렌드의 진원지인지**를 알아야 한다.
- 경쟁 채널 지형이 넓어지면 channel_dup·차별화 앵글 판단이 정밀해진다.

## 채택 절차
1. 딥리서치 결과 수신 → `/squeeze-report`로 사실성 분리(검증된 사실 vs 자기신고·추정).
2. **ACCEPT만** [[한국 AI 교육 유튜브 시장 구조]]에 본문화. 경쟁 채널 추가분은 벤치마크 표 확장.
3. 구독자 수치·자기신고 매출은 **인용 금지**(ADR-008/011 정합) — 포지션·포맷·훅 패턴만 채택.

## 면책 (CLAUDE.md 컴플라이언스 정합 의무)
- **구독자/트래픽 수치 영구거부:** ADR-008/011 전례. 변동성·교차검증 불완전·콘텐츠 불필요. 포지션·포맷만.
- **자기신고 수익·성과 수치 거부:** "월 N억" 류는 플랫폼 자기신고로 교차검증 불가 — overclaim_check 차단 대상. 인용 금지.
- **수익 단정·과장 금지:** 채널 소개에도 "이 채널 보면 무조건 수익" 류 단정 유도 금지.
- **단일 출처 수치 경계:** 원본 보고서가 한 출처(`[2]` 등)에만 의존한 수치(GitHub AI 코드 비율·생산성 %·CVE 번호)는 교차검증 전까지 "추정"으로 분류.
