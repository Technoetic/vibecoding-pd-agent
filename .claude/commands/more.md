현재 대화에서 다룬 파일·코드·문서·작업에 더 할 것이 없는지 점검한다.

**종료 조건: 추가 작업이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 탐색 에이전트 ≠ 판정 에이전트

같은 에이전트가 탐색하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[더 점검 1회차]`, `[더 점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (탐색 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 탐색 전문 에이전트다. 아래 대화 내용을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/더_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화에서 다룬 파일·코드·작업 목록을 여기에 삽입]

## 0단계: 작업 현황 요약
지금까지 완료된 작업과 다룬 대상을 3~5줄로 요약하라.
요약에서 빠진 항목이 있다면 즉시 보완하고 **추가 발생: YES**로 기록한다.

## 1단계: 작업 유형 분류
- 문서형 / 코드형 / 설정형 / 복합형 중 해당하는 것을 선택하라.

## 2단계: 미완료·누락 탐색 (선행 점검)
다음 관점으로 더 할 것을 탐색하라:

- **미완료**: 시작했지만 끝내지 않은 작업이 있는가?
- **파생 작업**: 완료한 작업에서 자연스럽게 이어져야 할 후속 작업이 있는가?
- **누락**: 해야 했는데 빠뜨린 항목이 있는가? (테스트, 동기화, 문서화 등)
- **일관성 깨짐**: A를 수정했는데 연관된 B는 수정하지 않아서 불일치가 생긴 경우가 있는가?

추가 항목이 발견되면 즉시 실행하고 **추가 발생: YES**로 기록한다.
추가 항목 1건당 -10점 패널티를 기록한다.

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 완결성 | 시작한 작업이 모두 완료되었는가? | 20점 |
| 일관성 | 연관 파일·섹션이 모두 함께 반영되었는가? | 20점 |
| 후속 처리 | 파생되는 작업이 누락 없이 처리되었는가? | 20점 |
| 동기화 | 프로젝트·글로벌 등 복수 위치가 동기화되었는가? | 20점 |
| 검증 가능성 | 완료 여부를 외부에서 확인할 수 있는 상태인가? | 20점 |

❌ 또는 ⚠️ 항목이 있으면 즉시 처리하고 **추가 발생: YES**로 기록한다.

- 0~3단계 전체 결과
- 추가 발생 여부: YES / NO
- 패널티 합계: -X점
- .claude/에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/더_점검_결과.md를 읽고 4단계만 수행하라. 탐색 내용을 수정하지 않는다.

## 4단계: 종합 결과
step_archive/더_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 패널티 - 항목별 감점) / 100점

### 이 회차에 추가 작업이 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 완료 상태를 한 문장으로 요약하라.
- 다른 사람에게 넘긴다면 추가로 설명할 사항이 있는가?
- 작업 유형에 맞는 다음 단계를 1가지 제안하라.
  - 문서형 → 실제 실행 시 동작 검증
  - 코드형 → 테스트 또는 리팩터링 방향
  - 설정형 → 적용 후 동작 확인
  - 복합형 → 가장 취약한 영역 우선 처리

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **추가 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **추가 발생 NO + 100점** → 종료한다.
