현재 대화에서 다룬 파일·코드·문서에 보강할 것이 없는지 점검한다.

**종료 조건: 보강 작업이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 보강 에이전트 ≠ 판정 에이전트

같은 에이전트가 보강하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[보강 점검 1회차]`, `[보강 점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (보강 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 보강 전문 에이전트다. 아래 대상 목록을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/보강_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화에서 다룬 파일·코드·문서 목록을 여기에 삽입]

## 0단계: 대상 요약
현재 대화에서 다룬 파일·코드·문서를 3~5줄로 요약하라.
요약에서 빠진 대상이 있다면 즉시 보완하고 **보강 발생: YES**로 기록한다.

## 1단계: 대상 유형 분류
- 문서형 / 코드형 / 데이터형 / 복합형 중 해당하는 것을 선택하라.

## 2단계: 보강 후보 감지 (선행 점검)
다음 관점으로 보강 후보를 탐색하라:
- 누락: 있어야 할 내용·섹션·경로가 빠져 있는가?
- 불완전: 절반만 구현되거나 설명이 끊긴 부분이 있는가?
- 불일치: 다른 파일·섹션과 내용이 어긋나거나 충돌하는가?
- 모호함: 에이전트나 독자가 다르게 해석할 수 있는 표현이 있는가?
보강 후보가 발견되면 즉시 보강하고 **보강 발생: YES**로 기록한다.
보강 후보 1건당 -10점 패널티를 기록한다.

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 완전성 | 필요한 내용이 빠짐없이 있는가? | 20점 |
| 일관성 | 다른 파일·섹션과 충돌 없이 일관된가? | 20점 |
| 명확성 | 에이전트·독자가 오해 없이 실행·이해할 수 있는가? | 20점 |
| 구조적 정합성 | 섹션 순서·위치·포맷이 올바른가? | 20점 |
| 응용 가능성 | 새로운 상황에서도 재사용·확장할 수 있는가? | 20점 |

❌ 또는 ⚠️ 항목이 있으면 즉시 보강하고 **보강 발생: YES**로 기록한다.

## 저장 형식
step_archive/보강_점검_결과.md에 다음을 저장하라:
- 0~3단계 전체 결과
- 보강 발생 여부: YES / NO
- 패널티 합계: -X점
- .claude/에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/보강_점검_결과.md를 읽고 4단계만 수행하라. 보강 내용을 수정하지 않는다.

## 4단계: 종합 결과
step_archive/보강_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 패널티 - 항목별 감점) / 100점

### 이 회차에 보강이 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 보강 완료 상태를 한 문장으로 요약하라.
- 이 대상을 다른 사람에게 넘긴다면 추가로 설명할 사항이 있는가?
- 대상 유형에 맞는 다음 단계를 1가지 제안하라.
  - 문서형 → 실제 실행 시 동작 검증
  - 코드형 → 테스트 또는 리팩터링 방향
  - 데이터형 → 유효성 검사 또는 활용 방향
  - 복합형 → 가장 취약한 영역 우선 보강 방향

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **보강 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **보강 발생 NO + 100점** → 종료한다.
