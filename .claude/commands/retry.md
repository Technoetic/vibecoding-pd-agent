현재 대화에서 "절대 못 한다"고 판단한 항목이 실제로 불가능한지 재검토한다.

**종료 조건: 재검토 후 추가로 시도할 방법이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 재검토 에이전트 ≠ 판정 에이전트

같은 에이전트가 재검토하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[못해 점검 1회차]`, `[못해 점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (재검토 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 재검토 전문 에이전트다. 아래 대화 내용을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/못해_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화에서 "불가능"으로 판단된 항목 목록을 여기에 삽입]

## 0단계: 불가 판단 항목 요약
"못 한다"고 결론 내린 항목을 3~5줄로 요약하라.
요약에서 빠진 항목이 있다면 즉시 보완하고 **재시도 발생: YES**로 기록한다.

## 1단계: 불가 판단 유형 분류
각 항목이 아래 중 어디에 해당하는지 판단하라:

- **기술적 한계** — 현재 도구·환경·권한으로 실제 불가능
- **정보 부족** — 더 많은 정보가 있으면 가능할 수도 있음
- **접근법 문제** — 다른 방법으로 시도하면 가능할 수도 있음
- **규칙 충돌** — 규칙/제약 때문에 막혔지만 규칙 자체를 재검토할 여지가 있음

## 2단계: 대안 탐색 (선행 점검)
포기하기 전에 반드시 다음을 점검하라:

- **우회 경로**: 직접 방법이 막혔을 때 간접적으로 같은 결과를 낼 수 있는가?
- **분해**: 한 번에 불가능하다면 더 작은 단위로 쪼개면 가능한가?
- **전제 재검토**: "불가능하다"는 판단의 전제 자체가 잘못된 것은 아닌가?
- **도구 전환**: 다른 도구·명령·접근법을 사용하면 가능한가?
- **부분 달성**: 100%는 불가능해도 80%는 가능한가? 그것으로 충분한가?

대안이 발견되면 즉시 시도하고 **재시도 발생: YES**로 기록한다.
대안 발견 1건당 -10점 패널티를 기록한다. (못 한다고 섣불리 포기한 것이므로)

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 진짜 불가 확인 | 모든 대안을 검토한 후에도 실제로 불가능한가? | 20점 |
| 전제 검증 | "불가능하다"는 전제 자체가 올바른가? | 20점 |
| 부분 달성 검토 | 완전 달성이 불가하면 부분 달성이라도 시도했는가? | 20점 |
| 대안 소진 | 시도 가능한 모든 접근법을 검토했는가? | 20점 |
| 한계 명시 | 왜 불가능한지 구체적 근거를 명시했는가? | 20점 |

⚠️ 또는 ❌ 항목이 있으면 해당 대안을 즉시 시도하고 **재시도 발생: YES**로 기록한다.

## 저장 형식
step_archive/못해_점검_결과.md에 다음을 저장하라:
- 0~3단계 전체 결과
- 재시도 발생 여부: YES / NO
- 패널티 합계: -X점
- .claude/commands/ 외 .claude/ 경로에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/못해_점검_결과.md를 읽고 4단계만 수행하라. 재검토 내용을 수정하지 않는다.

## 4단계: 종합 결과
못해_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 패널티 - 항목별 감점) / 100점

### 이 회차에 재시도가 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 진짜 불가능한 항목과 그 근거를 한 문장으로 요약하라.
- 불가능의 전제 조건이나 한계를 명시하라 (조건이 바뀌면 가능할 수 있는가?).
- 불가 유형에 맞는 다음 단계를 1가지 제안하라.
  - 기술적 한계 → 대체 가능한 유사 기능 제안
  - 정보 부족 → 어떤 정보가 있으면 가능한지 명시
  - 접근법 문제 → 다른 접근법 제안
  - 규칙 충돌 → 규칙 변경 또는 예외 적용 여부 사용자에게 확인

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **재시도 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **재시도 발생 NO + 100점** → 종료한다.
