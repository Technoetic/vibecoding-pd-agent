현재 대화에서 다룬 파일·코드·문서·작업의 흐름·순서·논리에 이상이 없는지 점검한다.

**종료 조건: 흐름 이상 항목이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 분석 에이전트 ≠ 판정 에이전트

같은 에이전트가 분석하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[흐름 점검 1회차]`, `[흐름 점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (흐름 분석 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 흐름 분석 전문 에이전트다. 아래 대화 내용을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/flow_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화에서 다룬 파일·코드·작업 흐름 목록을 여기에 삽입]

## 0단계: 흐름 현황 요약
현재 대화에서 다룬 흐름·순서·논리 구조를 3~5줄로 요약하라.
요약에서 빠진 흐름이 있다면 즉시 보완하고 **흐름 이상 발생: YES**로 기록한다.

## 1단계: 흐름 유형 분류
다룬 흐름이 아래 중 어디에 해당하는지 판단하라:

- **실행 흐름** — 코드·스크립트·명령의 실행 순서
- **데이터 흐름** — 입력→처리→출력의 데이터 이동 경로
- **제어 흐름** — 조건분기·반복·예외처리 구조
- **작업 흐름** — Step·Phase·에이전트 간 의존성과 순서
- **복합형** — 위 둘 이상 혼합

## 2단계: 흐름 이상 탐색 (선행 점검)
다음 관점으로 이상한 흐름을 탐색하라:

- **순서 역전**: 선행되어야 할 것이 후행되거나, 후행되어야 할 것이 선행된 경우
- **누락된 단계**: 흐름 중간에 빠진 단계가 있어 연결이 끊긴 경우
- **순환 의존**: A→B→A처럼 의존성이 순환하여 무한루프 또는 데드락이 발생하는 경우
- **불필요한 단계**: 흐름에 포함되어 있지만 실제로 아무 역할을 하지 않는 단계
- **분기 누락**: 조건분기에서 특정 경우의 수가 처리되지 않은 경우
- **조기 종료**: 흐름이 완료되기 전에 종료 조건이 발동되는 경우
- **맥락 불일치**: 이전 단계의 결과가 다음 단계의 입력과 맞지 않는 경우

흐름 이상이 발견되면 즉시 수정하고 **흐름 이상 발생: YES**로 기록한다.
흐름 이상 1건당 -10점 패널티를 기록한다.

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 순서 정합성 | 모든 단계가 올바른 순서로 배치되어 있는가? | 20점 |
| 연결 완전성 | 각 단계의 출력이 다음 단계의 입력과 일치하는가? | 20점 |
| 분기 완전성 | 모든 조건 분기가 처리되어 있는가? | 20점 |
| 종료 조건 | 흐름이 올바른 시점에 종료되는가? | 20점 |
| 의존성 비순환 | 순환 의존이 없고 단방향 흐름이 유지되는가? | 20점 |

❌ 또는 ⚠️ 항목이 있으면 즉시 수정하고 **흐름 이상 발생: YES**로 기록한다.

- 흐름 이상 발생 여부: YES / NO
- 패널티 합계: -X점
- .claude/commands/ 외 .claude/ 경로에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/flow_점검_결과.md를 읽고 4단계만 수행하라. 분석 내용을 수정하지 않는다.

## 4단계: 종합 결과
step_archive/flow_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 패널티 - 항목별 감점) / 100점

### 이 회차에 흐름 이상이 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 흐름이 정상임을 한 문장으로 요약하라.
- 흐름 보장의 전제 조건이나 한계가 있다면 명시하라.
- 흐름 유형에 맞는 다음 단계를 1가지 제안하라.
  - 실행 흐름 → 실제 실행 테스트
  - 데이터 흐름 → 경계값 및 예외 케이스 검증
  - 제어 흐름 → 분기별 시나리오 테스트
  - 작업 흐름 → 실제 에이전트 실행으로 검증
  - 복합형 → 가장 취약한 흐름 구간 우선 검증

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **흐름 이상 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **흐름 이상 발생 NO + 100점** → 종료한다.
