현재 대화에서 완료된 모든 작업·결과물을 100% 보장할 수 있는지 점검한다.

**종료 조건: 보장 불가 항목이 단 한 건도 없이 100점이 나온 회차에서만 종료한다.**

---

## 핵심 원칙: 검증 에이전트 ≠ 판정 에이전트

같은 에이전트가 검증하고 스스로 통과 판정하는 것을 **금지**한다. 반드시 독립된 두 서브에이전트를 순차 스폰한다.

---

## 매 회차 실행 절차

현재 몇 번째 점검인지 표시하라.
예: `[보장 점검 1회차]`, `[보장 점검 2회차]`, ...

### Phase 1: 에이전트 A 스폰 (검증 전문)

Agent 도구로 서브에이전트를 스폰한다. 프롬프트:

```
너는 검증 전문 에이전트다. 아래 작업 목록을 바탕으로 0~3단계를 수행하고, 결과를 step_archive/보장_점검_결과.md에 저장하라. 최종 점수 판정은 하지 않는다.

[대화에서 완료된 작업·결과물 목록을 여기에 삽입]

## 0단계: 작업 결과 요약
완료된 작업과 산출물을 3~5줄로 요약하라.
요약에서 빠진 항목이 있다면 즉시 보완하고 **보장 불가 발생: YES**로 기록한다.

## 1단계: 결과물 유형 분류
- 문서형 / 코드형 / 설정형 / 복합형 중 해당하는 것을 선택하라.

## 2단계: 보장 불가 항목 탐색 (선행 점검)
다음 관점으로 보장할 수 없는 항목을 탐색하라:

- **미검증**: 실제로 실행·확인하지 않고 "됐을 것"으로 넘어간 부분이 있는가?
- **가정 의존**: 특정 환경·조건이 맞아야만 동작하는데 그 조건을 확인하지 않았는가?
- **부작용 미확인**: 변경 사항이 다른 부분에 영향을 줬을 수 있는데 확인하지 않았는가?
- **재현 불가**: 동일한 절차를 다시 밟아도 같은 결과가 나올 것을 보장할 수 없는가?

보장 불가 항목이 발견되면 즉시 검증하거나 한계를 명시하고 **보장 불가 발생: YES**로 기록한다.
보장 불가 항목 1건당 -10점 패널티를 기록한다.

## 3단계: 항목별 점검
각 항목을 ✅ / ❌ / ⚠️(부분적)로 표시하라.

| 항목 | 기준 | 배점 |
|------|------|------|
| 실행 검증 | 실제로 실행·확인한 결과인가? | 20점 |
| 재현 가능성 | 동일 절차로 동일 결과를 재현할 수 있는가? | 20점 |
| 부작용 없음 | 변경이 의도치 않은 영향을 주지 않았는가? | 20점 |
| 조건 독립성 | 특정 환경·가정 없이도 동작하는가? | 20점 |
| 완전성 | 부분적 성공이 아닌 완전한 완료인가? | 20점 |

❌ 또는 ⚠️ 항목이 있으면 즉시 검증하거나 한계를 명시하고 **보장 불가 발생: YES**로 기록한다.

## 저장 형식
step_archive/보장_점검_결과.md에 다음을 저장하라:
- 0~3단계 전체 결과
- 보장 불가 발생 여부: YES / NO
- 패널티 합계: -X점
- .claude/에 파일 생성 금지
```

에이전트 A 완료를 기다린다.

### Phase 2: 에이전트 B 스폰 (판정 전문)

에이전트 A가 완료된 후, Agent 도구로 별도 서브에이전트를 스폰한다. 프롬프트:

```
너는 판정 전문 에이전트다. step_archive/보장_점검_결과.md를 읽고 4단계만 수행하라. 검증 내용을 수정하지 않는다.

## 4단계: 종합 결과
step_archive/보장_점검_결과.md의 내용을 독립적으로 검토하여 판정한다.

**종합 점수:** (기본 100점 - 패널티 - 항목별 감점) / 100점

### 이 회차에 보장 불가 항목이 한 건이라도 발생했는가?

**YES → 100점이어도 종료하지 않는다. 즉시 다음 회차 시작을 메인 에이전트에 지시하라.**

**NO + 100점 → 진짜 종료.**
- 보장 가능한 항목 전체를 한 문장으로 요약하라.
- 보장의 전제 조건이나 한계가 있다면 명시하라.
- 결과물 유형에 맞는 다음 단계를 1가지 제안하라.
  - 문서형 → 실제 실행 시 동작 검증
  - 코드형 → 테스트 커버리지 확인
  - 설정형 → 적용 후 동작 확인
  - 복합형 → 가장 불확실한 영역 우선 검증

**NO + 80~99점 → 다음 회차 시작을 메인 에이전트에 지시하라.**
**NO + 79점 이하 → 다음 회차 시작을 메인 에이전트에 지시하라.**

판정 결과를 출력하라.
```

### Phase 3: 반복 여부 결정

에이전트 B의 판정 결과를 확인한다.
- **보장 불가 발생 YES 또는 100점 미만** → 즉시 다음 회차를 시작한다.
- **보장 불가 발생 NO + 100점** → 종료한다.
