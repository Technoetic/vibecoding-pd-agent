# /domain-priorities — 프로젝트 지식·작업 우선순위 매트릭스

현재 프로젝트의 **결손 영역(gap)** — 미구현·TODO·문서화 안 된 지식·검증 안 된 가정 —을 자동 수집해 **5차원으로 점수화**하고, 가장 시급한 것부터 정렬한 **우선순위 매트릭스 1파일**을 영속화한다.

특정 도메인에 묶이지 않은 **범용 도구**다. 어떤 코드베이스/지식 베이스에도 적용된다 — 프로젝트의 실제 구조를 먼저 탐지한 뒤, 거기서 발견된 모듈·영역을 단위로 점수화한다.

## 사용법

```
/domain-priorities                          # 전체 결손 우선순위 매트릭스
/domain-priorities --top 10                 # 상위 10건만
/domain-priorities --area auth payments     # 특정 영역(모듈/디렉토리)만
/domain-priorities --src src/ engine/       # 소스 탐색 루트 지정
/domain-priorities --decisions docs/adr/    # 결정 기록(ADR/규율) 경로 지정
/domain-priorities --out reports/prio.md    # 출력 파일 경로 지정
/domain-priorities --gaps-only              # 점수화 없이 결손 목록만
/domain-priorities --refresh                # 기존 출력 덮어쓰기
```

인자가 없으면 모두 자동 추론한다(아래 단계 0 참조).

## 우선순위 산출 5 차원 (정량, 0~5점)

| 차원 | 의미 | 점수 기준 |
|---|---|---|
| **① 임팩트** | 이 결손이 사용자/비즈니스/프로젝트 목표에 미치는 영향 | 0=주변·5=핵심 목표·안전·신뢰 직결 |
| **② 노출 빈도** | 핵심 경로(hot path)·실 사용·반복 참조 빈도 | 0=거의 안 닿음·5=거의 매번 닿음 |
| **③ 구현 비용** | 이 AI 단독 진행 가능성 + 추정 소요 | 0=불가능/외부 결단 필요·5=즉시(≈1h) |
| **④ 근거 신뢰도** | 출처·검증 가능성(공식 문서·1차 자료·재현 가능) | 0=근거 없음/추정·5=공식·재현 검증됨 |
| **⑤ 위험도** | 방치 시 결함·실패·정책/제약 위반 가능성 | 0=무해·5=빌드 실패·실격·치명 결함 |

**우선순위 점수 = (① × 3) + (② × 3) + (④ × 2) + (⑤ × 2) − ③**  (③은 비용이므로 감점)
**최대 점수**: 73점

→ 점수 높을수록 우선 처리. 점수 0 이하 → DEFER.

> 프로젝트 성격에 따라 차원 의미를 맥락에 맞게 해석하라. 점수 기준은 고정이되, "임팩트"가 무엇인지는 프로젝트 목표(예: 마감·용량·매출·안전)에서 끌어온다.

## 행동 절차

### 단계 0 — 프로젝트 구조 탐지 + 결손 수집

먼저 **실제 구조를 탐지**한다(가정 금지):

1. **결정 기록 위치 자동 탐지** — `--decisions` 없으면 흔한 경로를 순서대로 탐색: `docs/adr/`, `**/decisions/`, `**/rules/`, `ADR-*.md`, `CLAUDE.md`, `00-meta/`. 있으면 그 frontmatter·"한계/제약/TODO" 절을 읽는다. **없으면 이 출처는 건너뛴다**(에러 아님).
2. **소스/모듈 영역 자동 탐지** — `--src` 없으면 리포 루트의 주요 디렉토리(소스·엔진·패키지)를 Glob으로 식별해 영역 단위로 삼는다.
3. **결손 마커 수집** — 탐지된 영역에서 `TODO`/`FIXME`/`placeholder`/`stub`/`미구현`/`HACK`/빈 함수/`NotImplemented`를 Grep. 빈 디렉토리·미작성 노트·깨진 링크도 결손.
4. **세션 누적 결손** — 이번 대화에서 드러난 ⚠ 결손·검증 실패·"나중에" 항목을 포함.

수집 후 중복 제거 + 영역(area) 분류. **추정으로 결손을 만들지 말 것** — 실제 마커·실제 파일·실제 결정 기록에 근거.

### 단계 1 — 5차원 자동 점수화

각 결손에 5차원 점수 부여:

- **임팩트**: 프로젝트의 명시된 목표(마감/품질/용량/안전 등)에 비추어 평가.
- **노출 빈도**: 핵심 경로·진입점·반복 참조 여부.
- **구현 비용**: 이 AI 단독 가능 여부 — 외부 결단(사업·법무·인프라·유료 구매·키 발급)이 필요하면 비용 높음(점수 낮음).
- **근거 신뢰도**: 필요 시 WebFetch/공식 문서로 라이브 검증. 검증 불가/추정이면 0 + "근거 없음" 명시.
- **위험도**: 결정 기록·정책·제약(있다면) 위반 위험 + 방치 시 결함 심각도.

### 단계 2 — 우선순위 매트릭스 생성

```markdown
## 결손 우선순위 (<date> 기준)

### 🔴 즉시 진행 (점수 ≥ 50)
| 순위 | 영역 | 결손 | 점수 | AI 단독? |
|---|---|---|---|---|
| 1 | <area> | <gap> | 58 | ✅ / ⚠ 외부 의존 |

### 🟡 단기 진행 (점수 30~49)
...

### 🟢 데이터/조건 충족 후 (점수 10~29)
...

### ⏸ 사용자 결단 보류 (점수 0~9)
...
```

### 단계 3 — md 영속화

출력 경로(`--out` > 자동: `reports/`·`docs/`·`vault/reports/` 중 존재하는 곳 > 없으면 리포 루트)에 `domain-priorities-<YYYY-MM-DD>.md` 작성:

```yaml
---
type: domain_priorities
generated: <YYYY-MM-DD>
generated_by: /domain-priorities
total_gaps: <N>
immediate: <N>
short_term: <N>
deferred: <N>
scanned_areas:
  - <auto-detected area>
methodology: 5_dimension_scoring
---
```

본문 6절 구조:
1. **우선순위 매트릭스** (🔴 즉시 · 🟡 단기 · 🟢 조건 충족 후 · ⏸ 보류)
2. **상위 N건 정밀 분석** (각 결손별 임팩트·위험·진행 방안)
3. **AI 단독 진행 가능 영역** (즉시 착수 가능 목록)
4. **사용자 결단 영역 분리** (🔵 사업·법무·인프라·유료 구매·외부 키)
5. **조건/데이터 의존 영역** (특정 조건·실측·트래픽 누적 후 측정)
6. **다음 단계 추천** (사용자에게 진행 신호 요청)

### 단계 4 — 사용자 보고

```
## /domain-priorities 결과 (<date>)

상위 5 결손 영역 (즉시 진행 권장):
1. 🔴 <area> <gap> (58점) — AI 단독 가능
2. ...

전체 <N> 결손 영역 → <출력 경로>
AI 단독 진행 가능: <N>건 / 사용자 결단 영역: <N>건 / 조건 충족 후: <N>건

다음 단계 권장: 1순위 진행 신호 시 즉시 착수.
```

## 안전장치

### 오추정 차단 (의무)

점수화 전 반드시:
- 탐지된 소스/모듈 영역 실제 Read·Glob·Grep
- 결정 기록(있으면) frontmatter·제약 절 직접 읽기
- 세션 ⚠ 결손 보고 검토
- **추정 "결손" 금지** — 실 코드·실 마커·실 결정 기록 근거 의무. 구조가 없으면 "해당 출처 없음"으로 명시하고 건너뛴다(날조 금지).

### 가짜 근거 차단

④ 근거 신뢰도 점수 부여 시 라이브 검증:
- 인용한 문서/논문/스펙 URL 실재 확인(WebFetch)
- 검증 실패 → 점수 0 + "근거 미검증" 명시. 출처를 지어내지 말 것.

### 사용자 결단 영역 분리

🔵 사업(UI·가격·결제·마케팅)·법무·인프라·유료 구매·외부 API 키는 AI 단독 진행 X → "사용자 결단 보류"로 분리하고 점수화는 하되 착수 신호는 사용자에게 넘긴다.

### 결정 기록·정책 정합 (있을 때만)

프로젝트에 결정 기록(ADR)·규율 문서·정책이 있으면 각 결손의 위반 위험을 ⑤ 위험도에 반영한다. **없으면 이 검증은 생략한다** — 존재하지 않는 ADR을 가정하지 말 것.

## 활용 시나리오

- **개발 우선순위 결단**: `/domain-priorities --top 5` → 상위 5건 + AI 단독 가능 영역 즉시 신호.
- **결손 목록만**: `/domain-priorities --gaps-only` → 점수화 없이 결손 + 사용자 결단 영역 분리.
- **특정 영역 정밀**: `/domain-priorities --area <a> <b>` → 해당 영역만 매트릭스.

## /domain-priorities 특성

| 항목 | 내용 |
|---|---|
| 출력 | 횡단 우선순위 **1파일** |
| 용도 | 결손 우선순위 결단(의사결정용) |
| 빈도 | 개발 단계 변경·로드맵 재검토 시 |
| 사용자 결단 영역 | 명시 + 점수화(착수는 사용자 신호) |
| AI 단독 진행 신호 | 우선순위 + 즉시 착수 가능 영역 명시 |

## 출력 파일 위치

- `<출력 경로>/domain-priorities-<YYYY-MM-DD>.md` — 일자별 영속화
- 인접 `INDEX.md`가 있으면 갱신(선택)

## 주의

- 본 명령어는 **합리적 우선순위만 산출**. 사업·외부 결단 영역은 별도 분리.
- 모든 점수는 호출 시점 기준 — 시간 경과·구조 변경 시 재호출 권장.
- 프로젝트에 결정 기록·정책이 있으면 그 정합을 ⑤에 반영하되, 없으면 가정하지 말 것.
- 결손을 추정·날조하지 말 것. 실제 구조에 근거가 없으면 "근거 없음"으로 보고.
